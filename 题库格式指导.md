## 二、标准字段清单（必须字段 + 可选字段）

| 字段名 | 类型 | 是否必须 | 说明 | 示例值 |
|--------|------|----------|------|--------|
| `id` | Number | ✅ | 题目唯一ID，递增整数 | `1`, `2`, `3` |
| `difficulty` | String | ✅ | 难度等级：`beginner` / `intermediate` / `advanced` | `'beginner'` |
| `street` | String | ✅ | 牌局阶段：`preflop` / `flop` / `turn` / `river` | `'flop'` |
| `position` | String | ✅ | 玩家位置：`UTG`, `UTG+1`, `MP`, `MP+1`, `CO`, `BTN`, `SB`, `BB`, `IP`(有利位置), `OOP`(不利位置) | `'BTN'` |
| `tableSize` | Number | ✅ | 桌型人数：6或8 | `8` |
| `hand` | Array | ✅ | 玩家手牌，用Unicode花色：♠♥♦♣ | `['A♠', 'K♠']` |
| `board` | Array | ✅ | 公共牌，翻前为空数组 | `['K♠', '7♦', '2♣']` |
| `potSize` | Number | ✅ | 当前底池大小（单位：bb） | `9` |
| `effectiveStack` | Number | ✅ | 有效筹码深度（单位：bb） | `100` |
| `currentBet` | Number | ✅ | 当前需要跟注的下注量（单位：bb） | `3` |
| `actionToPlayer` | String | ✅ | 轮到玩家前的对手动作描述 | `'BTN open to 3bb'` 或 `''`(首个行动) |
| `optimalAction` | String | ✅ | GTO最优动作：`fold` / `check` / `call` / `bet` / `raise` | `'raise'` |
| `actionFrequency` | Number | ✅ | 该动作在GTO策略中的执行频率（0-1） | `1.0`, `0.65` |
| `ev` | Number | ✅ | 该动作的期望值（单位：bb） | `0.85` |
| `potOdds` | Number | ✅ | 底池赔率（跟注需要的胜率），无下注时为0 | `2.5` |
| `mdf` | Number | ✅ | 最小防守频率（%），无下注时为0 | `40` |
| `explanation` | String | ✅ | 一句话结论性解释（显示在前端GTO反馈标题） | `'AKs是顶级起手牌，在按钮位应该100%率先加注。'` |
| `reasoning` | String | ✅ | 详细推理分析（支持`\n`换行，显示在前端推理分析区） | `'位置优势+牌力强...'` |
| `equity` | String | ✅ | 对抗对手范围的胜率估算 | `'52-54%'`, `'38%'` |
| `opponentRange` | Object | ✅ | 对手范围：value(价值牌) + bluff(诈唬牌)，无时为`null` | `{ value: 'TT+, AK', bluff: 'KQs, JTs' }` |
| `teaching` | Array | ✅ | 教学要点列表（显示在前端教学要点区） | `['位置优势...', '起手牌强度...']` |
| `commonMistakes` | Array | ✅ | 常见错误列表（显示在前端教学要点区） | `['错误：平跟溜入...']` |
| `tags` | Array | ✅ | 标签列表，用于分类和检索 | `['preflop', 'open', 'BTN', 'AK']` |

## 三、从iflow知识库生成题目的映射规则

### 3.1 资料来源 → 字段映射

| 资料类型 | 对应字段 | 提取方法 |
|----------|----------|----------|
| 《Modern Poker Theory》章节 | `reasoning`, `teaching` | 提取核心论点，总结成中文 |
| 6-Max Cash GTO Ranges PDF | `hand`, `position`, `optimalAction`, `actionFrequency` | 按位置+手牌查表，填对应动作和频率 |
| GTO Wizard Blog 牌面分析 | `board`, `street`, `opponentRange`, `mdf`, `potOdds` | 按牌面描述提取公共牌、对手范围、计算频率 |
| 《扑克数学》公式章节 | `ev`, `equity`, `potOdds` | 按公式计算期望值、胜率、底池赔率 |
| 知乎/德扑专栏汉化稿 | `explanation`, `commonMistakes` | 提取结论性语句和常见错误 |
| Solver Reports | `optimalAction`, `actionFrequency`, `ev` | 直接引用Solver输出结果 |

### 3.2 生成流程（iflow → 题目）

```
1. iflow语义检索 → 输入关键词："翻牌前 BTN位置 AKs 最佳策略"
2. 提取检索结果 → 定位到《Modern Poker Theory》第X章 + GTO Wizard对应报告
3. 映射到字段 → 按上表填写JSON对象
4. 校验 → 检查必填字段是否齐全、数值是否合理（频率0-1、胜率格式正确）
5. 追加到scenarios.js → 下一个ID递增，插入SCENARIOS数组
6. 测试 → 运行游戏，验证题目显示和GTO反馈是否正常
```

### 3.3 标签体系（标准化）

**阶段标签**：`preflop`, `flop`, `turn`, `river`  
**动作标签**：`open`, `defend`, `3bet`, `4bet`, `check`, `call`, `bet`, `raise`, `fold`  
**位置标签**：`UTG`, `MP`, `CO`, `BTN`, `SB`, `BB`, `IP`, `OOP`  
**牌型标签**：`AK`, `QQ`, `77`, `suited`, `offsuit`, `pocketpair`, `broadway`  
**概念标签**：`position`, `potodds`, `MDF`, `setmine`, `semibluff`, `balance`, `boardtexture`, `equity`, `rangeconstruction`  
**来源标签**：`MPT`(Modern Poker Theory), `POP`(Play Optimal Poker), `TMOP`(The Mathematics of Poker), `GTOWizard`, `LiuLiAo`(刘立奥)

## 四、录入规范

1. **ID管理**：新题目ID = 当前最大ID + 1，严禁重复
2. **花色符号**：统一使用 ♠ ♥ ♦ ♣（Unicode），禁止使用 s/h/d/c 缩写
3. **数值精度**：`actionFrequency`保留两位小数，`ev`保留两位小数，`potOdds`保留两位小数
4. **字符串格式**：`reasoning`中用 `\n` 换行，`teaching`和`commonMistakes`用数组，每项不超过80字
5. **范围描述**：`opponentRange`用扑克标准记法，如 `TT+`(TT及以上), `AKs`(同花AK), `KJo`(异色KJ)
6. **难度判定**：
   - 初级：翻前开放/防守、基础底池赔率计算
   - 中级：翻后决策、MDF应用、混合策略
   - 高级：不利位置策略、薄价值下注、复杂范围构建
