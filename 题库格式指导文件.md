## 二、遇到的问题及修复方案

### 问题1：f-string语法错误（反斜杠问题）
**现象**：Python f-string中不能直接包含反斜杠，导致生成脚本失败
**修复方案**：
- 改用字符串拼接（`reasoning = "根据GTO策略，" + hand_name + "在"...`）
- 避免f-string中的反斜杠转义
- 使用`replace()`处理字符串转义

### 问题2：牌面分析算法粗糙，误判牌面纹理
**现象**：
- A♠ K♦ 2♣ 被误判为"湿润牌面（连张听牌）"
- 实际是干燥牌面（高牌主导，无同花/顺子听牌）

**修复方案**：
- 重写`analyze_board_texture()`函数
- 精确判断：同花听牌需要≥3张同花色，顺子听牌需要牌面连续
- 干燥牌面标准：不成对 + 无同花听牌 + 无顺子听牌

### 问题3：手牌评估错误，未正确识别听牌
**现象**：
- J♠ T♠ 在 A♠ K♦ 2♣ 牌面有同花听牌（3张♠）
- 被错误评估为"未命中牌面，可能是最弱牌"
- 实际是同花听牌，约35%胜率

**修复方案**：
- 重写`evaluate_hand_on_board()`函数
- 正确计算：同花听牌（max_suit_count >= 3）、顺子听牌（牌面连续）
- 区分：顶对、中间对、双听牌、单听牌、未命中

### 问题4：下注建议与牌面类型不匹配
**现象**：
- 干燥牌面（如A-high）显示"下注大小建议：2/3-满池（湿润牌面）"
- 湿润牌面显示干燥牌面的下注建议

**修复方案**：
- 批量修复脚本`fix_bet_size_final.py`
- 根据牌面类型（dry/wet）匹配正确的下注建议
- 干燥牌面：1/2-2/3底池；湿润牌面：2/3-满池

### 问题5：翻后推理简化，缺乏实质内容
**现象**：
- 早期生成的翻后推理只有一句话："翻后推理：根据GTO策略..."
- 缺乏牌面纹理分析、手牌强度评估、GTO频率依据

**修复方案**：
- 生成详细推理结构（5段式）：
  1. 牌面纹理分析
  2. 手牌强度评估
  3. GTO频率依据（引用GTO Wizard Blog）
  4. 具体策略（含下注大小建议）
  5. 翻后计划
- 使用知识库内容填充每段

### 问题6：翻前题数量异常（已修复）
**现象**：
- 预期：翻前题200+道（ID 1-200左右）
- 实际：只剩33道（ID 1-33）
- ID 34+的street字段被错误改为翻后（flop/turn/river）

**修复方案**：
- 重新生成470道翻前题（ID 501-970）
- 翻前题总数恢复到503道（33原+470新）
- 题库从500题扩充到970题

### 问题7："未命中牌面"错误描述（已修复）
**现象**：
- 282处手牌明明有听牌或成牌，却被描述为"未命中牌面"
- 例如：JTs在A♠ K♦ 2♣有同花听牌，却被标记为"未命中"

**修复方案**：
- 运行`fix_missed_board.py`脚本
- 根据手牌在牌面的实际强度，替换正确描述
- 修复后残留"未命中牌面"：0处（除了真正未命中的牌）
